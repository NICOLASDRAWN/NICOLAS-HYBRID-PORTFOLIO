  FRONTEND (pages/CreateOrder.tsx):
  ─────────────────────────────────────────
  
  Validaciones:
  • Tipo de archivo: application/pdf o extensión .pdf
  • Tamaño máximo: 10MB (10 * 1024 * 1024 bytes)
  • Estado de carga: isUploadingDoc
  
  Estados visuales:
  • Normal: border-slate-200 hover:border-blue-400
  • Cargando: border-blue-500 bg-blue-50 cursor-wait
  
  Toast notifications:
  • Loading: toast.loading('📤 Subiendo documento PDF...')
  • Success: toast.success con nombre del archivo
  • Error: toast.error con descripción detallada

  BACKEND (backend/server.js):
  ─────────────────────────────────────────
  
  Endpoint: POST /api/upload/documents
  
  Validaciones:
  • Tipo MIME: req.file.mimetype === 'application/pdf'
  • Extensión: file.originalname.toLowerCase().endsWith('.pdf')
  • Tamaño: req.file.size <= 10MB
  
  Acciones en error:
  • Eliminar archivo subido: fs.removeSync()
  • Retornar error 400 con detalles
  • Log del error en consola
  
  Logs:
  • Inicio: "📂 Iniciando subida de archivo... Tipo: documents"
  • Validación: "✅ PDF validado correctamente: nombre.pdf (123KB)"
  • Éxito: "✅ Archivo guardado exitosamente: /uploads/..."
  • Error: "❌ Archivo rechazado: nombre.doc (tipo: application/msword)"

┌────────────────────────────────────────────────────────────────────────────┐
│  🚀 FLUJO DE CARGA MEJORADO                                               │
└────────────────────────────────────────────────────────────────────────────┘

  1. Usuario selecciona archivo
     ↓
  2. Validación en navegador (tipo y tamaño)
     ↓
  3. Si válido: Mostrar indicador de carga
     ↓
  4. Enviar archivo al servidor
     ↓
  5. Validación en servidor (tipo, tamaño)
     ↓
  6. Si válido: Guardar archivo
     ↓
  7. Retornar URL del archivo
     ↓
  8. Agregar a lista de documentos
     ↓
  9. Mostrar mensaje de éxito
     ↓
  10. Limpiar input y ocultar indicador

  Si hay error en cualquier paso:
  • Mostrar mensaje de error específico
  • Limpiar input
  • Ocultar indicador de carga
  • Eliminar archivo temporal (si fue subido)

│  🧪 CÓMO PROBAR                                                           │
└────────────────────────────────────────────────────────────────────────────┘

  TEST 1: Subir archivo PDF válido
  ─────────────────────────────────
  1. Crear nueva orden
  2. Ir a Paso 3
  3. Click en "Vincular Nuevo Archivo PDF"
  4. Seleccionar archivo .pdf (menor a 10MB)
  5. Ver spinner "Subiendo PDF..."
  6. Ver mensaje: "✅ Documento PDF vinculado correctamente"
  7. Ver archivo en la lista

  TEST 2: Intentar subir archivo no-PDF
  ──────────────────────────────────────
  1. Click en "Vincular Nuevo Archivo PDF"
  2. Seleccionar archivo .docx o .jpg
  3. Ver mensaje: "❌ Solo se permiten archivos PDF"
  4. Archivo no se sube

  TEST 3: Intentar subir PDF muy grande
  ──────────────────────────────────────
  1. Click en "Vincular Nuevo Archivo PDF"
  2. Seleccionar archivo .pdf mayor a 10MB
  3. Ver mensaje: "❌ Archivo demasiado grande"
  4. Ver tamaño del archivo en el mensaje
  5. Archivo no se sube

  TEST 4: Error de conexión
  ─────────────────────────
  1. Detener el backend
  2. Intentar subir archivo PDF
  3. Ver mensaje: "❌ Error al subir documento"
  4. Ver descripción: "Verifica tu conexión..."
